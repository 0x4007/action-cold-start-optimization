# Introduction

Welcome to the WebAssembly-Optimized GitHub Actions SDK documentation. This section will help you get started with building high-performance GitHub Actions plugins using our WebAssembly-optimized SDK.

## What You'll Learn

In this Getting Started section, you'll learn how to:

1. **Install** the necessary tools and dependencies
2. **Understand** the basic concepts of our SDK
3. **Create** your first plugin with a step-by-step quick start guide
4. **Set up** a development environment for efficient plugin development

## Prerequisites

Before you begin, you should have:

- Basic knowledge of JavaScript
- Familiarity with GitHub Actions
- Node.js (version 16 or later) installed
- Git installed

## No WebAssembly Knowledge Required

One of the key benefits of our SDK is that you don't need to understand WebAssembly to get started. The SDK provides abstractions that allow you to build high-performance plugins using familiar JavaScript, while leveraging WebAssembly under the hood for performance-critical operations.

## Development Tiers

Our SDK supports three development tiers, allowing you to choose the approach that matches your skill level and performance requirements:

- **Tier 1: JavaScript-Only** - Perfect for beginners or those who want to get started quickly
- **Tier 2: TypeScript + WASM Integration** - For those who want more type safety and control
- **Tier 3: Full Stack with Rust** - For maximum performance customization

In this Getting Started guide, we'll focus on Tier 1 (JavaScript-Only) to help you get up and running quickly. Later sections of the documentation cover the more advanced tiers.
